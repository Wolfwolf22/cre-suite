# CRE Suite вЂ” Commercial Real Estate Intelligence Platform

A full-stack commercial real estate web application with 6 AI-powered tools for deal analysis, document generation, and financial modeling.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + Tailwind CSS + Vite |
| Backend | Node.js + Express |
| Database | PostgreSQL via Prisma ORM |
| Auth | Clerk (email + Google OAuth) |
| AI | OpenAI (`gpt-4o-mini`) |
| PDF Export | React-PDF / @react-pdf/renderer |
| Charts | Recharts |

---

## Prerequisites

- Node.js 18+
- PostgreSQL 14+ running locally (or a hosted DB)
- Accounts with: Clerk, OpenAI, ATTOM Data (optional), US Census Bureau (optional)

---

## Setup Instructions

### 1. Clone and navigate

```bash
cd "cre-suite"
```

### 2. Configure environment variables

Edit the `.env` file in the root of `cre-suite/`:

```env
OPENAI_API_KEY=sk-...                 # Required вЂ” get from platform.openai.com
DATABASE_URL=postgresql://...          # Required вЂ” your PostgreSQL connection string
CLERK_SECRET_KEY=sk_...               # Required вЂ” get from clerk.com dashboard
VITE_CLERK_PUBLISHABLE_KEY=pk_...     # Required вЂ” get from clerk.com dashboard
ATTOM_API_KEY=...                     # Optional вЂ” property data (attomdata.com)
CENSUS_API_KEY=...                    # Optional вЂ” census demographics (api.census.gov/data/key_signup.html)
PORT=3001                             # Optional вЂ” defaults to 3001
```

Also copy the Clerk publishable key to `client/.env`:

```env
VITE_CLERK_PUBLISHABLE_KEY=pk_...
```

### 3. Set up the database

```bash
cd server
npx prisma db push       # Creates all tables from schema
npx prisma generate      # Generates the Prisma client
```

### 4. Install dependencies

```bash
# Server
cd server && npm install

# Client
cd ../client && npm install
```

### 5. Run locally

**Terminal 1 вЂ” Backend:**
```bash
cd server && npm run dev
# Server starts on http://localhost:3001
```

**Terminal 2 вЂ” Frontend:**
```bash
cd client && npm run dev
# App opens on http://localhost:5173
```

## Environment Variable Guide

| Variable | Required | Source | Description |
|---|---|---|---|
| `OPENAI_API_KEY` | Yes | platform.openai.com | Powers all 6 AI tools (gpt-4o-mini) |
| `DATABASE_URL` | Yes | Your PostgreSQL instance | `postgresql://user:pass@host:5432/dbname` |
| `CLERK_SECRET_KEY` | Yes | clerk.com dashboard | Server-side Clerk auth |
| `VITE_CLERK_PUBLISHABLE_KEY` | Yes | clerk.com dashboard | Client-side Clerk auth |
| `ATTOM_API_KEY` | No | attomdata.com | Property owner, sale history, assessed value |
| `CENSUS_API_KEY` | No | api.census.gov/data/key_signup.html | Census tract demographics |
| `PORT` | No | вЂ” | Server port (default: 3001) |

### What works without optional API keys

- ATTOM not configured: Property Intelligence shows geocoding, flood zone, and census data only
- Census not configured: Property Intelligence shows geocoding, ATTOM data, and flood zone only
- All other tools (LOI, Cash Flow, Debt, Lease, Deal Analyzer) work fully with only `OPENAI_API_KEY`

## The 6 Tools

### Tool 1 вЂ” LOI Generator (`/tools/loi`)
- Fill in buyer, seller, property, price, and terms
- Claude drafts a complete professional Letter of Intent
- Copy to clipboard or export to PDF
- Auto-saves to linked deal

### Tool 2 вЂ” Property Intelligence (`/tools/property`)
- Enter any US address
- Pulls from ATTOM Data, Census Bureau, OpenStreetMap Nominatim, FEMA flood hazard layer
- Claude writes a 1-paragraph property narrative
- Shows owner, value, flood zone, and demographics

### Tool 3 вЂ” Cash Flow Analyzer (`/tools/cashflow`)
- Input purchase price, financing, income, expenses, and rent growth
- Calculates NOI, cap rate, cash-on-cash, DSCR, GRM, 5-year IRR, equity multiple
- Shows a 5-year projection table + Recharts line chart
- Claude writes a 2-sentence deal summary
- Export to PDF

### Tool 4 вЂ” Debt Screener (`/tools/debt`)
- Input NOI, price, property type, LTV, term, amortization
- Calculates max loan, DSCR, debt yield
- Claude scores the deal: Green/Yellow/Red risk rating
- Shows qualifying lender channels (Bridge, Agency, CMBS, SBA 504, Life Co, Local Bank)
- Generates a 3-bullet lender pitch

### Tool 5 вЂ” Lease Generator (`/tools/lease`)
- Input all lease terms: parties, type (NNN/Gross/Modified Gross), rent, escalation, TI, CAM, etc.
- Claude drafts a complete commercial lease with all standard clauses
- Copy to clipboard or export to PDF

### Tool 6 вЂ” Deal Analyzer (`/tools/deal`)
- Input address, type, price, NOI, target market
- Claude researches market cap rates, comparable sales, vacancy trends
- Outputs a 1вЂ“10 deal score broken down across 4 categories (color-coded)
- Full investment memorandum generated by AI
- Export to PDF

## Rate Limits

- AI endpoints: 10 requests per user per hour
- General API: 200 requests per user per 15 minutes

## Clerk Setup

1. Create a Clerk application at [clerk.com](https://clerk.com)
2. Enable Email + Google OAuth sign-in methods
3. Copy the **Publishable Key** to `client/.env` as `VITE_CLERK_PUBLISHABLE_KEY`
4. Copy the **Secret Key** to `.env` as `CLERK_SECRET_KEY`
5. The app uses custom headers (`x-clerk-user-id`, `x-clerk-user-email`) for API auth вЂ” no webhook setup required for local dev

---

## Production Deployment Notes

- Set `NODE_ENV=production` and `CLIENT_URL=https://yourdomain.com` in server env
- The `uploads/` folder is local by default; swap `multer` storage for S3 using the `AWS_*` env vars (architecture is S3-ready)
- Run `npx prisma migrate deploy` instead of `prisma db push` in production
- Build the client: `cd client && npm run build` вЂ” serve `dist/` as static files or deploy to Vercel/Netlify
